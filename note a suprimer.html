problaime 1 : Énoncé du problème Lorsque l'utilisateur clique sur un nom de filtre (par exemple “Mariage”, “Portrait”, ou “Tous”), le filtre s'applique bien les photos s'affichent correctement mais le bouton du filtre actif ne change pas de couleur. L'utilisateur ne voit donc pas quel filtre est sélectionné. Origine du problème Le plugin mauGallery gère déjà la logique de filtrage (cacher/montrer les images selon le tag), mais il n’ajoutait pas systématiquement la classe CSS indiquant quel bouton est actif. La classe CSS .active-tag existe bien dans le code (elle change la couleur du bouton actif). Cependant, aucune fonction JavaScript ne mettait à jour cette classe au clic. Résultat : le style du bouton actif ne s’affichait jamais. Solution apportée Ajouté une fonction JavaScript appelée setupGalleryFilters(). Elle écoute les clics sur chaque bouton de filtre et met à jour leur apparence. function setupGalleryFilters() { const filterButtons = document.querySelectorAll('.tags-bar .nav-link'); filterButtons.forEach(button => { button.addEventListener('click', () => { // Enlève la classe active-tag à tous les boutons filterButtons.forEach(btn => btn.classList.remove('active-tag')); //️⃣ Ajoute la classe active-tag au bouton cliqué button.classList.add('active-tag'); }); }); } document.addEventListener('DOMContentLoaded', setupGalleryFilters); Problaime 2; Quand on cliquais sur les flèches « < » ou « > » dans la lightbox (la fenêtre d’image agrandie), l’image ne changeait pas. Pourquoi : Le souci venait du code des deux fonctions : prevImage() (image précédente) nextImage() (image suivante) Dans ces fonctions, le code disait en gros ceci : next = imagesCollection[index] || imagesCollection[0]; $(".lightboxImage").attr("src", $(next).attr("src")); Cela voulait dire : « Montre l’image à la position actuelle » au lieu de dire « Montre l’image suivante ou précédente ». Résultat : Quand on cliquais sur une flèche, le script rechargeait la même image, donc rien ne bougeait à l’écran. La cause exacte : Le développeur du plugin n’a jamais changé la variable index avant d’afficher l’image suivante. Il ne faisait que reprendre le même numéro de l’image actuelle. C’est une erreur logique, pas une erreur de syntaxe. Ssolution : Pour la flèche gauche (précédente) : prevImage() avent imagesCollection[index] || imagesCollection[imagesCollection.length - 1]; $(".lightboxImage").attr("src", $(next).attr("src")); apres let prevIndex = index === 0 ? imagesCollection.length - 1 : index - 1; $(".lightboxImage").attr("src", $(imagesCollection[prevIndex]).attr("src")); si on est sur la première image, on revient à la dernière sinon, on va à l’image juste avant. Pour la flèche droite (suivante) : nextImage() avent next = imagesCollection[index] || imagesCollection[0]; $(".lightboxImage").attr("src", $(next).attr("src")); Aller à l'image suivante (en boucle) apres let nextIndex = (index + 1) % imagesCollection.length; $(".lightboxImage").attr("src", $(imagesCollection[nextIndex]).attr("src")); on passe à l’image suivante → si on est à la dernière, on revient à la première.
